API ENDPOINTS
=============

Authentication (Public)
  POST   /api/auth/signup               - Register new user
  POST   /api/auth/login                - Login user
  POST   /api/auth/refresh              - Refresh JWT token

Patient Routes (Protected - Role: patient)
  POST   /api/patients/onboarding/save-draft       - Save draft
  GET    /api/patients/onboarding/draft            - Get draft
  POST   /api/patients/onboarding/submit           - Submit onboarding
  GET    /api/patients/profile                     - Get profile
  GET    /api/patients/doctor                      - Get assigned doctor

Doctor Routes (Protected - Role: doctor)
  GET    /api/doctors/patients                     - List assigned patients
  GET    /api/doctors/patients/:patientId          - Get patient details
  GET    /api/doctors/profile                      - Get doctor profile

Health Check
  GET    /health                        - Server health status


TECHNOLOGIES
============
- Express.js - Web framework
- JWT (jsonwebtoken) - Authentication tokens
- bcryptjs - Password hashing
- Socket.io - Real-time messaging
- Supabase - PostgreSQL database
- CORS - Cross-origin resource sharing
- dotenv - Environment configuration


DATABASE SCHEMA HIGHLIGHTS
==========================
✓ User authentication with password hashing
✓ Role-based access (patient/doctor)
✓ Patient profiles with onboarding data
✓ Doctor profiles with specialization
✓ Patient-doctor assignments
✓ Draft saving for multi-step forms
✓ Chat messages with read status
✓ Chat rooms for session management
✓ Proper foreign keys & constraints
✓ Performance indexes on frequently queried fields


ENVIRONMENT VARIABLES
=====================
PORT - Server port (default: 5000)
NODE_ENV - Environment (development/production)
FRONTEND_URL - Frontend URL for CORS

JWT_SECRET - Secret key for JWT signing
JWT_EXPIRY - Token expiration (default: 7d)

SUPABASE_URL - Supabase project URL
SUPABASE_KEY - Supabase anonymous key
SUPABASE_SERVICE_ROLE_KEY - Service role for admin operations
DATABASE_URL - PostgreSQL connection string


QUICK START
===========
1. npm install
2. Copy .env.example to .env and fill in Supabase credentials
3. Run SQL from database/schema.sql in Supabase
4. npm run dev
5. Server runs on http://localhost:5000
